# Flamingo UI

Frontend for Flamingo Downloader, built with React, TypeScript, Vite, Ant Design, and the Tauri 2 webview runtime.

This folder contains the desktop UI only. Rust/Tauri backend code lives in `../src` and `../src-tauri`.

## Prerequisites

- Node.js 20.19+ recommended
- npm

Vite 7 prints a warning on older Node 20.x releases. If you see a version warning during `npm run build`, upgrade Node before changing build behavior.

## Commands

Install dependencies:

```bash
npm install
```

Run the UI dev server only:

```bash
npm run dev
```

Lint:

```bash
npm run lint
```

Unit tests:

```bash
npm run test:unit
```

Production build:

```bash
npm run build
```

Bundle budget check:

```bash
npm run check:bundle
```

## Important Scripts

- `build`: i18n checks, TypeScript build, then Vite production build
- `check:i18n`: validates locale keys/placeholders and regenerates typed i18n keys
- `check:bundle`: validates generated bundle sizes against project budgets
- `test:unit`: runs Vitest unit tests

## Entry Points

- `src/main.tsx`: main desktop window
- `src/logs.tsx`: operation logs window
- `index.html`: main window HTML entry
- `logs.html`: logs window HTML entry

## Structure

- `src/App.tsx`: main application shell and orchestration
- `src/api/`: Tauri invoke client
- `src/components/`: reusable UI pieces and dialogs
- `src/pages/`: page-level shells for lazy-loaded areas
- `src/hooks/`: local hooks such as table layout persistence
- `src/stores/`: small UI state stores
- `src/i18n/`: locale dictionaries and generated key typing
- `src/utils/`: formatting, shortcut parsing, and small helpers

## Build Notes

- Heavy dialogs are lazy-loaded to keep the initial entry chunk smaller.
- Vite manual chunking keeps `react`, `tauri`, and large Ant Design payloads stable enough for CI bundle checks.
- `ui/dist/bundle-size-report.json` is generated by `npm run check:bundle` and is the artifact used by CI for budget enforcement.

## Maintenance Guidelines

- Keep user-visible strings in `src/i18n/*.json`; do not hardcode new UI copy in components.
- Prefer focused dialog/page components over growing `src/App.tsx`.
- If a UI change increases bundle size, run `npm run build` and `npm run check:bundle` before merging.
- When adding shortcuts, update both the shortcut utilities and their tests.
